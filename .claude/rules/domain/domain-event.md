# 领域事件（Domain Event）开发规范

## 核心设计原则

领域事件记录**已发生的业务事实**，用于聚合之间的通信和最终一致性协调。

### 事件命名

- **过去时态**：`UserCreated`、`OrderPlaced`、`PaymentProcessed`
- **不是命令**：❌ `CreateUser`、`PlaceOrder` 是命令

### 事件产生

在聚合根的业务方法中产生事件

### 事件发布

Application Handler 负责发布事件

### 事件处理

其他聚合通过事件处理器监听和处理事件


## 必须事项
- 定义在 `YuG.Domain/Events/` 目录
- 实现 `IDomainEvent` 接口或继承 `DomainEvent` 基类
- 使用不可变类型规范（见 code-style.md）
- **必须包含**：触发事件的聚合根 ID
- **可以包含**：相关的值对象和关键数据
- **禁止包含**：实体引用、导航属性、其他聚合的状态


## 禁止事项

- **禁止命令式命名**（`CreateUser`），事件是已发生的事实
- **禁止在聚合产生事件时同时处理**（在 Handler 中处理）
- **禁止在事件中包含实体对象**，只包含 ID 和值对象
- **禁止阻塞式事件发布**，应异步处理
- **禁止事件处理中修改多个聚合**（一个处理器处理一个聚合）
- **禁止在事件处理器中抛出异常导致数据库回滚**（应有补偿逻辑）
- **禁止跳过事件发布**，聚合产生的所有事件都应被发布
